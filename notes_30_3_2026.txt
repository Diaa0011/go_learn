	---about concurrency in go---
	- goroutine isn't a task or process as in c# all go routines run in the same task or process
	- channel is something like way of communications between different processes 
	- select is for handling channels it stops the code execution until one of cases can run
	- select is only random among cases that are ready at the exact same time if they are not ready it will be temporal readiness
